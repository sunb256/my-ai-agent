はい、**「Tool結果から決定的にChartSpecを作る」は基本的にルールベース**です。ただし、LLMを完全に排除するという意味ではなく、役割を分けます。

**おすすめは、Toolがデータを作り、バックエンドがルールでChartSpecを作り、final_answer tool call はその成果物を参照するだけにする設計**です。final_answer にグラフJSONを直接作らせると、せっかく安定化のために final_answer tool call にしているのに、そこがまた不安定になります。

全体像はこうです。

```txt
User
  ↓
LLM
  ↓
ToolCall: query_sales_data
  ↓
ToolResult: DataFrameArtifact を返す
  ↓
Backend: DataFrameArtifact から ChartSpec をルール生成
  ↓
LLM
  ↓
final_answer tool call
  ├─ message: 最終回答文
  └─ artifact_refs: ["chart_xxx"]
  ↓
Frontend
  ├─ message を表示
  └─ chart_xxx を ChartRenderer で表示
```

重要なのは、**final_answer が ChartSpec 本体を生成しない**ことです。final_answer は「このチャートを表示してよい」という参照だけ持ちます。

例えば ToolResult はこうします。

```py
from typing import Literal, Any
from pydantic import BaseModel, Field


class ColumnSpec(BaseModel):
    key: str
    label: str
    kind: Literal["temporal", "category", "number", "text"]
    unit: str | None = None


class DataFrameArtifact(BaseModel):
    type: Literal["dataframe"] = "dataframe"
    artifact_id: str
    title: str
    columns: list[ColumnSpec]
    rows: list[dict[str, Any]]
    display_policy: Literal["auto", "table", "chart", "chart_and_table", "none"] = "auto"


class ToolResultPayload(BaseModel):
    text: str
    artifacts: list[DataFrameArtifact] = Field(default_factory=list)
```

売上取得ツールなら、こういう結果を返します。

```py
def query_sales_data() -> ToolResultPayload:
    df = DataFrameArtifact(
        artifact_id="sales_monthly_001",
        title="月別売上",
        columns=[
            ColumnSpec(key="month", label="月", kind="temporal"),
            ColumnSpec(key="sales", label="売上", kind="number", unit="円"),
        ],
        rows=[
            {"month": "2026-01", "sales": 1200000},
            {"month": "2026-02", "sales": 1350000},
            {"month": "2026-03", "sales": 1280000},
        ],
        display_policy="auto",
    )

    return ToolResultPayload(
        text="月別売上データを取得しました。",
        artifacts=[df],
    )
```

この `DataFrameArtifact` を受け取った直後に、バックエンド側で ChartSpec を作ります。

```py
class ChartSeries(BaseModel):
    dataKey: str
    label: str
    valueSuffix: str | None = None


class ChartSpec(BaseModel):
    type: Literal["chart"] = "chart"
    artifact_id: str
    source_artifact_id: str
    title: str
    chartType: Literal["line", "bar", "pie", "scatter"]
    xKey: str
    series: list[ChartSeries]
    data: list[dict[str, Any]]
```

ルールは最初は単純で十分です。

```py
def build_chart_spec(df: DataFrameArtifact) -> ChartSpec | None:
    if df.display_policy == "none":
        return None

    temporal_cols = [c for c in df.columns if c.kind == "temporal"]
    category_cols = [c for c in df.columns if c.kind == "category"]
    number_cols = [c for c in df.columns if c.kind == "number"]

    if not number_cols:
        return None

    # 時系列 + 数値 => 折れ線
    if temporal_cols:
        x_col = temporal_cols[0]
        y_col = number_cols[0]

        return ChartSpec(
            artifact_id=f"chart_{df.artifact_id}",
            source_artifact_id=df.artifact_id,
            title=df.title,
            chartType="line",
            xKey=x_col.key,
            series=[
                ChartSeries(
                    dataKey=y_col.key,
                    label=y_col.label,
                    valueSuffix=y_col.unit,
                )
            ],
            data=df.rows,
        )

    # カテゴリ + 数値 => 棒グラフ
    if category_cols:
        x_col = category_cols[0]
        y_col = number_cols[0]

        return ChartSpec(
            artifact_id=f"chart_{df.artifact_id}",
            source_artifact_id=df.artifact_id,
            title=df.title,
            chartType="bar",
            xKey=x_col.key,
            series=[
                ChartSeries(
                    dataKey=y_col.key,
                    label=y_col.label,
                    valueSuffix=y_col.unit,
                )
            ],
            data=df.rows,
        )

    return None
```

つまり、**LLMに「どう描くか」を任せない**。
Toolが返した列メタデータを見て、バックエンドが決定的に変換します。

そして final_answer tool call はこうします。

```py
class FinalArtifactRef(BaseModel):
    artifact_id: str
    view: Literal["chart", "table", "auto"] = "auto"


class FinalAnswerArgs(BaseModel):
    message: str
    artifact_refs: list[FinalArtifactRef] = Field(default_factory=list)
```

LLMが最後に呼ぶ final_answer は、この程度にします。

```json
{
  "message": "月別売上を見ると、2月に増加し、3月にやや減少しています。",
  "artifact_refs": [
    {
      "artifact_id": "chart_sales_monthly_001",
      "view": "chart"
    }
  ]
}
```

ここでのポイントは、`artifact_id` が存在するかどうかをバックエンドで検証することです。存在しないIDを final_answer が指定したら無視するか、フォールバックでテキストだけ返します。

実装上は、`ctx.artifacts` のような置き場を作るとよいです。

```py
class ExecContext(BaseModel):
    exec_id: str
    artifacts: dict[str, Any] = Field(default_factory=dict)
```

Tool実行後に登録します。

```py
def register_tool_artifacts(ctx: ExecContext, result: ToolResultPayload) -> None:
    for artifact in result.artifacts:
        ctx.artifacts[artifact.artifact_id] = artifact

        if artifact.type == "dataframe":
            chart = build_chart_spec(artifact)
            if chart is not None:
                ctx.artifacts[chart.artifact_id] = chart
```

final_answer 後に、参照された artifact を解決します。

```py
def build_final_response(ctx: ExecContext, final: FinalAnswerArgs) -> dict:
    resolved_artifacts = []

    for ref in final.artifact_refs:
        artifact = ctx.artifacts.get(ref.artifact_id)
        if artifact is not None:
            resolved_artifacts.append(artifact)

    return {
        "message": final.message,
        "artifacts": resolved_artifacts,
    }
```

```tsx
function ArtifactRenderer({ artifact }: { artifact: any }) {
  if (artifact.type === "chart") {
    return <ChartRenderer spec={artifact} />;

  if (artifact.type === "dataframe") {

  }

  return <JsonRenderer value={artifact} />;
```

final_answer tool call を使っている場合、悩むのは「ToolResultの時点でチャートを表示するか」「final_answer後に表示するか」です。おすすめは次のどちらかです。

| 方式               | 内容                               | 向いているケース    |
| ---------------- | -------------------------------- | ----------- |
| final_answer後に表示 | final_answer が参照した artifact だけ表示 | 安定重視        |
| ToolResult時点で仮表示 | Tool実行後すぐチャート表示し、final_answerで説明 | ストリーミングUI重視 |

今の設計で安定性を重視しているなら、まずは **final_answer後に表示** がよいです。つまり、ToolResultでは `ctx.artifacts` に保存するだけで、ユーザーにはまだ出さない。最後に final_answer が `artifact_refs` で参照したものだけを返します。

ただし、LLMが `artifact_refs` を指定し忘れる可能性があります。その対策として、バックエンド側に `auto_show` を持たせると安定します。

```py
class DataFrameArtifact(BaseModel):
    type: Literal["dataframe"] = "dataframe"
    artifact_id: str
    title: str
    columns: list[ColumnSpec]
    rows: list[dict[str, Any]]
    display_policy: Literal["auto", "table", "chart", "chart_and_table", "none"] = "auto"
    auto_show: bool = True
```

そして final_answer が artifact を指定しなかった場合でも、`auto_show=True` の artifact を自動添付できます。

```py
def build_final_response(ctx: ExecContext, final: FinalAnswerArgs) -> dict:
    selected_ids = {ref.artifact_id for ref in final.artifact_refs}

    if not selected_ids:
        selected_ids = {
            artifact_id
            for artifact_id, artifact in ctx.artifacts.items()
            if getattr(artifact, "auto_show", False)
        }

    return {
        "message": final.message,
        "artifacts": [
            artifact
            for artifact_id, artifact in ctx.artifacts.items()
            if artifact_id in selected_ids
        ],
    }
```

この設計にすると、final_answer は今のまま「安定した最終回答ツール」として使えます。変更点は、final_answer のスキーマに `artifact_refs` を追加するだけです。

```py
@tool(name="final_answer")
def final_answer(
    message: str,
    artifact_refs: list[FinalArtifactRef] | None = None,
) -> FinalAnswerArgs:
    return FinalAnswerArgs(
        message=message,
        artifact_refs=artifact_refs or [],
    )
```

最終的な責務分担はこう考えると分かりやすいです。

```txt
Tool
  業務データを取得・集計する

Backend Artifact Builder
  DataFrameArtifact から ChartSpec をルール生成する

LLM
  最終説明文を書く
  どの artifact を回答に添えるか選ぶ

final_answer tool
  message と artifact_refs を構造化して返す

Frontend
  artifact.type に応じて固定コンポーネントで描画する
```

この形なら、final_answer tool call の安定性を維持したまま、グラフUIも実現できます。特に大事なのは、**グラフの元データと描画仕様は ctx.artifacts に保存し、final_answer はそれを参照するだけにする**ことです。これで、LLMが数値を壊したり、存在しない系列を作ったりするリスクをかなり下げられます。
