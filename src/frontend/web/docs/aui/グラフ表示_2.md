その指摘は正しいです。**チャート表示は「単体のUI」ではなく、LLMの解説文とセットで1つの回答として扱うべき**です。

なので設計としては、`final_answer` の返り値を単なる `message + artifacts` ではなく、**「文章ブロック」と「artifact参照ブロック」を並べた回答構造**にするのがよいです。

assistant-ui 側では backend tool の result を React renderer で描画できるので、`final_answer` tool の result を `FinalAnswerView` に渡し、その中で「文章 → チャート → 補足コメント」のように表示します。assistant-ui の tool rendering は、tool の実行場所に関係なく renderer を登録できる設計です。([assistant-ui][1]) また、既存backend toolに対してUIだけを登録する使い方も想定されています。([assistant-ui][2])

一番おすすめはこの形です。

```txt id="x3zd3f"
final_answer
  └─ blocks
      ├─ text
      ├─ artifact_ref(chart)
      ├─ text
      └─ artifact_ref(table)
```

つまり、`final_answer` のスキーマをこうします。

```py id="m9pcui"
from typing import Literal, Annotated
from pydantic import BaseModel, Field


class TextBlock(BaseModel):
    type: Literal["text"] = "text"
    text: str


class ArtifactBlock(BaseModel):
    type: Literal["artifact"] = "artifact"
    artifact_id: str
    view: Literal["auto", "chart", "table"] = "auto"
    caption: str | None = None


AnswerBlock = Annotated[
    TextBlock | ArtifactBlock,
    Field(discriminator="type"),
]


class FinalAnswerArgs(BaseModel):
    blocks: list[AnswerBlock]
```

`FinalAnswerView` はこの `blocks` を順番に描画します。

```tsx id="qig84v"
import { ArtifactRenderer } from "@/components/agent-ui/artifact-renderer";

type TextBlock = {
  type: "text";
  text: string;
};

type ArtifactBlock = {
  type: "artifact";
  artifact: unknown;
  caption?: string | null;
};

type FinalAnswerResult = {
  blocks: Array<TextBlock | ArtifactBlock>;
};

type Props = {
  result: unknown;
};

export function FinalAnswerView({ result }: Props) {
  const final = result as FinalAnswerResult;

  return (
    <div className="space-y-4">
      {final.blocks.map((block, index) => {
        if (block.type === "text") {
          return (
            <div key={index} className="whitespace-pre-wrap leading-7">
              {block.text}
            </div>
          );
        }

        if (block.type === "artifact") {
          return (
            <div key={index} className="space-y-2">
              <ArtifactRenderer artifact={block.artifact} />

              {block.caption ? (
                <div className="text-sm text-muted-foreground">
                  {block.caption}
                </div>
              ) : null}
            </div>
          );
        }

        return null;
      })}
    </div>
  );
}
```

こうすると、表示はこういう構造になります。

```txt id="kgqpk5"
[LLMコメント]
月別売上は2月に増加し、3月にやや減少しています。

[チャート]
月別売上グラフ

[LLMコメント]
3月の減少要因は、キャンペーン終了や主要顧客の発注タイミングと合わせて確認するとよいです。
```

ここで重要なのは、**ChartSpecはbackendがルール生成し、解説文はLLMが生成する**という役割分担です。

```txt id="sgky8m"
backendが決定的に作るもの:
  - ChartSpec
  - data
  - xKey
  - series
  - chartType

LLMが作るもの:
  - 要約
  - 着目点
  - 注意点
  - 次に見るべき観点
  - artifactをどの位置に挿入するか
```

つまり、LLMにチャートのデータは作らせません。しかし、**チャートをどう説明するか** はLLMに任せます。

実装上は `final_answer` をこう変えるのがよいです。

```py id="9rvnrh"
@tool(name="final_answer")
def final_answer(blocks: list[AnswerBlock]) -> FinalAnswerArgs:
    return FinalAnswerArgs(blocks=blocks)
```

そして backend 側で `ArtifactBlock` を解決します。

```py id="oqltjb"
def resolve_final_answer(ctx: ExecContext, final: FinalAnswerArgs) -> dict:
    resolved_blocks = []

    for block in final.blocks:
        if block.type == "text":
            resolved_blocks.append(block.model_dump())
            continue

        if block.type == "artifact":
            artifact = ctx.artifacts.get(block.artifact_id)

            if artifact is None:
                continue

            resolved_blocks.append({
                "type": "artifact",
                "artifact": artifact.model_dump(),
                "caption": block.caption,
            })

    return {
        "blocks": resolved_blocks,
    }
```

この設計なら、1つの回答内に複数のチャートと複数のコメントを混ぜられます。

```txt id="bq8jvv"
text:
  "売上は増加傾向です"

chart:
  月別売上

text:
  "ただし利益率は下がっています"

chart:
  月別利益率

text:
  "売上増加に対して利益率が落ちているので、原価か値引きの影響を確認してください"
```

単純な `message + artifacts` 方式だと、どうしても「文章が上、チャートが下」に固定されます。最初はそれでもよいですが、分析UIとして自然なのは **blocks方式** です。

最小実装としては、段階を分けるとよいです。

```txt id="gnfnzx"
第1段階:
  final_answer = { message, artifacts }
  文章の下にチャートを出す

第2段階:
  final_answer = { blocks }
  文章とチャートを任意順序で混ぜる

第3段階:
  chartごとに caption / insight / warning を付ける
```

あなたの用途だと、第2段階の `blocks` 方式まで最初から入れてよいと思います。`final_answer tool call` を使っているなら、むしろこのほうが自然です。最終回答そのものを「テキストだけ」ではなく、**テキストとUI成果物の並び**として安定化できるからです。

最終的な形はこうです。

```txt id="6s5bzn"
LLM:
  final_answer(blocks=[...]) を呼ぶ

backend:
  blocks内の artifact_id を ctx.artifacts から解決する

assistant-ui:
  final_answer の backend tool renderer を呼ぶ

FinalAnswerView:
  text block は文章表示
  artifact block は ChartRenderer / TableRenderer で表示
```

この設計なら、チャートは単なる飾りではなく、**LLMの解説の途中に埋め込まれる分析結果**になります。
